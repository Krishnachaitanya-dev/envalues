## Architecture Overview

**AlaChat** is a WhatsApp chatbot builder SPA (no SSR) built on:
- **React 18 + TypeScript + Vite** (SWC for fast compilation)
- **Supabase** for auth, database, and edge functions
- **TanStack React Query** for server state
- **shadcn-ui** (Radix UI primitives) + **Tailwind CSS** for UI
- **Razorpay** for subscription payments

### Routing (`src/App.tsx`)
React Router v6 with a flat route list. The `/dashboard/*` subtree uses `DashboardLayout` which wraps all dashboard pages in `DashboardProvider` + sidebar/topbar shell.

### State Management
- **Server state**: React Query (`useDashboardData.ts` hook is the main data layer — handles auth, chatbot configs, QA pairs, WhatsApp setup, and subscriptions)
- **Dashboard UI state**: `DashboardContext` (`src/contexts/DashboardContext.tsx`) shared via `DashboardProvider`
- **Forms**: React Hook Form + Zod schemas (validation schemas live alongside the hook that uses them)

### Key Files
- `src/hooks/useDashboardData.ts` — central data management hook; all Supabase reads/writes go through here
- `src/integrations/supabase/client.ts` — Supabase client (reads `VITE_SUPABASE_URL` and `VITE_SUPABASE_PUBLISHABLE_KEY`)
- `src/integrations/supabase/types.ts` — auto-generated DB types (do not hand-edit)
- `src/layouts/DashboardLayout.tsx` — auth guard + dashboard shell
- `src/components/dashboard/builder/` — visual chat flow builder components
- `supabase/migrations/` — database migrations
- `supabase/functions/` — edge functions

### Path Alias
`@/*` resolves to `src/*` (configured in `vite.config.ts` and `tsconfig.json`).

### Environment Variables
Stored in `.env`:
```
VITE_SUPABASE_URL
VITE_SUPABASE_PUBLISHABLE_KEY
VITE_SUPABASE_PROJECT_ID
```

### Design System
- Dark theme with WhatsApp green (`#25D366`) accents
- Fonts: Syne (display), DM Sans (body)
- shadcn-ui components live in `src/components/ui/` — generated by the shadcn CLI, extend rather than modify
- Tailwind dark mode is class-based (`dark:` prefix)

### Testing
Vitest with jsdom environment. Setup file at `src/test/setup.ts`. Tests use Testing Library React globals.
